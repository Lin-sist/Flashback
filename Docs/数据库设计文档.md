# 《时光回序》数据库设计文档（MVP）

本文档用于固定 MVP 第一版的数据库结构，约束字段、索引、枚举和值类型，减少 GitHub Copilot 在建表、实体类、Mapper 和 SQL 生成过程中的随意发挥。

---

## 1. 设计目标

数据库设计只服务于当前 MVP，支持以下主链路：

`创建草稿 -> 编辑草稿 -> 封存记录 -> 到期解锁 -> 回看记录 -> 写一句回信`

本版本数据库设计遵循以下原则：

1. 优先支持核心业务，避免过度设计
2. 所有状态字段使用固定字符串枚举
3. 列表查询和定时解锁相关字段必须建索引
4. 标签、AI、后台只保留 MVP 所需扩展位

---

## 2. 数据库环境约定

### 2.1 数据库版本
- `MySQL 8.x`

### 2.2 字符集与排序规则
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`

### 2.3 命名规范
- 表名使用小写下划线风格
- 字段名使用小写下划线风格
- 主键统一使用 `id`
- 时间字段统一使用 `created_at`、`updated_at`
- 状态字段统一使用 `varchar` 存储固定枚举值，不使用数字魔法值

### 2.4 主键规范
- 主键类型统一建议为 `BIGINT`
- 主键生成方案可选：
  - 数据库自增
  - 雪花算法

为降低第一版复杂度，建议先使用数据库自增主键。

- `DISABLED`

### 3.2 记录状态 `record.status`

## 5. 详细表结构设计

## 5.1 `user`

用途：存储普通用户信息。

| 字段名 | 类型 | 非空 | 默认值 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| `id` | `bigint` | 是 | 自增 | 主键 |
| `username` | `varchar(50)` | 是 | 无 | 用户名，唯一 |
| `password_hash` | `varchar(255)` | 是 | 无 | 密码哈希 |
| `nickname` | `varchar(50)` | 是 | 无 | 昵称 |
| `email` | `varchar(100)` | 否 | `NULL` | 邮箱 |
| `avatar` | `varchar(255)` | 否 | `NULL` | 头像地址 |
| `status` | `varchar(20)` | 是 | `ENABLED` | 用户状态 |
| `created_at` | `datetime` | 是 | `CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | `datetime` | 是 | `CURRENT_TIMESTAMP` | 更新时间 |

索引设计：
- 主键：`PRIMARY KEY (id)`
- 唯一索引：`uk_user_username (username)`

说明：
- 第一版建议用户名登录
- `email` 可选，不强制唯一

---

## 5.2 `record`

用途：存储用户记录主体，是业务核心表。

| 字段名 | 类型 | 非空 | 默认值 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| `id` | `bigint` | 是 | 自增 | 主键 |
| `user_id` | `bigint` | 是 | 无 | 所属用户 ID |
| `title` | `varchar(100)` | 否 | `NULL` | 标题，可为空 |
| `content` | `text` | 是 | 无 | 记录正文 |
| `record_type` | `varchar(30)` | 是 | `NODE_RECORD` | 记录类型 |
| `core_question` | `varchar(255)` | 否 | `NULL` | 当下最在意的问题 |
| `status` | `varchar(20)` | 是 | `DRAFT` | 记录状态 |
| `unlock_at` | `datetime` | 否 | `NULL` | 解锁时间，封存时必填 |
| `sealed_at` | `datetime` | 否 | `NULL` | 封存时间 |
| `unlocked_at` | `datetime` | 否 | `NULL` | 解锁完成时间 |
| `ai_summary` | `text` | 否 | `NULL` | AI 整理结果 |
| `ai_prompt_result` | `text` | 否 | `NULL` | AI 写作提示结果 |
| `created_at` | `datetime` | 是 | `CURRENT_TIMESTAMP` | 创建时间 |
| `updated_at` | `datetime` | 是 | `CURRENT_TIMESTAMP` | 更新时间 |

索引设计：
- 主键：`PRIMARY KEY (id)`
- 普通索引：`idx_record_user_id (user_id)`
- 普通索引：`idx_record_status (status)`
- 普通索引：`idx_record_unlock_at (unlock_at)`
- 组合索引：`idx_record_user_status_created (user_id, status, created_at)`
- 组合索引：`idx_record_status_unlock_at (status, unlock_at)`

说明：
- `unlock_at` 在草稿阶段可为空
- 进入 `SEALED` 状态后，`unlock_at` 必须非空
- 回看阶段依赖 `unlocked_at`

## 5.4 `record_tag`

| 字段名 | 类型 | 非空 | 默认值 | 说明 |

## 6. 表关系说明

### 6.1 关系图
- 一个 `user` 可以拥有多条 `record`
- 一条 `record` 可以绑定多个 `tag`
- 一个 `tag` 可以被多条 `record` 使用
- 一条 `record` 只能有一条 `reply`
- 一条 `record` 可以对应多条 `unlock_notice_log`

### 6.2 逻辑外键

第一版可在代码层维护关系，不强制数据库物理外键。

建议逻辑关系如下：
- `record.user_id -> user.id`
- `record_tag.record_id -> record.id`
- `record_tag.tag_id -> tag.id`
- `reply.record_id -> record.id`
- `reply.user_id -> user.id`
- `unlock_notice_log.record_id -> record.id`
- `unlock_notice_log.user_id -> user.id`

说明：
- 如果项目是课程项目，逻辑外键更灵活，便于调试
- 如果团队明确要求数据库外键，也可以后续补充

## 7. 状态流转对应的数据变化

### 7.1 新建草稿
- `record.status = DRAFT`
- `unlock_at = NULL` 或由用户先填写
- `sealed_at = NULL`
- `unlocked_at = NULL`

### 7.2 编辑草稿
- 允许更新：
  - `title`
  - `content`
  - `record_type`
  - `core_question`
  - `unlock_at`
  - 标签关联

### 7.3 封存记录
- `record.status` 从 `DRAFT` 改为 `SEALED`
- `unlock_at` 必须非空且大于当前时间
- 更新 `sealed_at`

### 7.4 自动解锁
- 仅处理 `status = SEALED` 且 `unlock_at <= 当前时间`
- 更新为 `status = UNLOCKED`
- 写入 `unlocked_at`
- 插入一条 `unlock_notice_log`

### 7.5 创建回信
- 校验 `record.status = UNLOCKED`
- 校验 `reply` 表中当前 `record_id` 不存在记录
- 插入 `reply`

## 9. 建表 SQL 参考

以下 SQL 为 MVP 参考版本，便于直接交给 Copilot 继续生成实体类和 Mapper。

```sql
CREATE TABLE `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL,
  `password_hash` VARCHAR(255) NOT NULL,
  `nickname` VARCHAR(50) NOT NULL,
  `email` VARCHAR(100) DEFAULT NULL,
  `avatar` VARCHAR(255) DEFAULT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'ENABLED',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_user_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE `record` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `title` VARCHAR(100) DEFAULT NULL,
  `content` TEXT NOT NULL,
  `record_type` VARCHAR(30) NOT NULL DEFAULT 'NODE_RECORD',
  `core_question` VARCHAR(255) DEFAULT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'DRAFT',
  `unlock_at` DATETIME DEFAULT NULL,
  `sealed_at` DATETIME DEFAULT NULL,
  `unlocked_at` DATETIME DEFAULT NULL,
  `ai_summary` TEXT DEFAULT NULL,
  `ai_prompt_result` TEXT DEFAULT NULL,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_record_user_id` (`user_id`),
  KEY `idx_record_status` (`status`),
  KEY `idx_record_unlock_at` (`unlock_at`),
  KEY `idx_record_user_status_created` (`user_id`, `status`, `created_at`),
  KEY `idx_record_status_unlock_at` (`status`, `unlock_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE `tag` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `name` VARCHAR(50) NOT NULL,
  `type` VARCHAR(20) NOT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'ENABLED',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_tag_name_type` (`name`, `type`),
  KEY `idx_tag_type` (`type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE `record_tag` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `record_id` BIGINT NOT NULL,
  `tag_id` BIGINT NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_record_tag` (`record_id`, `tag_id`),
  KEY `idx_record_tag_record_id` (`record_id`),
  KEY `idx_record_tag_tag_id` (`tag_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE `reply` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `record_id` BIGINT NOT NULL,
  `user_id` BIGINT NOT NULL,
  `content` VARCHAR(500) NOT NULL,
  `reply_type` VARCHAR(20) NOT NULL DEFAULT 'SHORT_REPLY',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_reply_record_id` (`record_id`),
  KEY `idx_reply_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE `unlock_notice_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `record_id` BIGINT NOT NULL,
  `user_id` BIGINT NOT NULL,
  `notice_type` VARCHAR(30) NOT NULL DEFAULT 'SYSTEM_UNLOCK',
  `notice_status` VARCHAR(20) NOT NULL DEFAULT 'SUCCESS',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_unlock_notice_record_id` (`record_id`),
  KEY `idx_unlock_notice_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

CREATE TABLE `admin_user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL,
  `password_hash` VARCHAR(255) NOT NULL,
  `nickname` VARCHAR(50) NOT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'ENABLED',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_admin_user_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
