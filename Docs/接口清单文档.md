# 《时光回序》接口清单文档（MVP）

本文档用于固定 MVP 第一版的接口边界、请求参数、响应结构和权限规则，目标是让 GitHub Copilot 在生成 Controller、DTO、VO、Service 时尽量少返工。

---

## 1. 文档目标

本文档只解决以下问题：

1. 当前 MVP 需要哪些接口
2. 每个接口的请求参数和返回字段是什么
3. 每个接口的权限边界和业务校验是什么
4. 列表接口的分页和筛选方式如何统一
5. Copilot 生成接口代码时必须遵守哪些规范

---

## 2. 通用规范

### 2.1 接口前缀
- 用户端接口前缀：`/api`
- 管理端接口前缀：`/admin`

### 2.2 认证方式
- 用户端：`Authorization: Bearer <token>`
- 管理端：`Authorization: Bearer <admin_token>`

### 2.3 数据格式
- 请求体：`application/json`
- 时间格式：`yyyy-MM-dd HH:mm:ss`

### 2.4 统一返回体

建议统一返回体结构如下：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

说明：
- `code = 0` 表示成功
- 非 `0` 表示失败
- `message` 用于简要说明
- `data` 为具体业务数据

### 2.5 分页结构

列表接口统一返回：

```json
{
  "list": [],
  "total": 0,
  "pageNum": 1,
  "pageSize": 10
}
```

### 2.6 错误码建议

| 错误码 | 含义 |
| ---- | ---- |
| `0` | 成功 |
| `4001` | 参数错误 |
| `4002` | 业务状态不允许 |
| `4003` | 记录不存在 |
| `4004` | 标签不存在 |
| `4005` | 回信已存在 |
| `4010` | 未登录或 token 无效 |
| `4030` | 无权限访问 |
| `5000` | 系统异常 |
| `5001` | AI 服务暂不可用 |

## 3. 用户端接口

## 3.1 注册

### 接口
- `POST /api/auth/register`

### 请求体

```json
{
  "username": "lin",
  "password": "123456",
  "nickname": "Lin"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| `username` | `string` | 是 | 用户名，唯一 |
| `password` | `string` | 是 | 明文密码，服务端加密存储 |
| `nickname` | `string` | 是 | 昵称 |

### 响应体

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userId": 1
  }
}
```

### 业务规则
- 用户名不能重复
- 密码不允许明文入库

---

## 3.2 登录

### 接口
- `POST /api/auth/login`

### 请求体

```json
{
  "username": "lin",

}
```

### 响应体

```json
{
  "code": 0,
```

## 4. 记录模块接口

## 4.1 创建草稿

### 接口
- `POST /api/records`

### 请求体

```json
{
  "title": "给三个月后的自己",
  "content": "最近在准备实习，有点迷茫。",
  "recordType": "NODE_RECORD",
  "coreQuestion": "我能找到合适的实习吗",
  "unlockAt": "2026-06-01 09:00:00",
  "tagIds": [1, 6]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| `title` | `string` | 否 | 标题 |
| `content` | `string` | 是 | 正文 |
| `recordType` | `string` | 是 | `FUTURE_LETTER` / `NODE_RECORD` / `EMOTION_NOTE` |
| `coreQuestion` | `string` | 否 | 当前最在意的问题 |
| `unlockAt` | `string` | 否 | 草稿阶段可先不填 |
| `tagIds` | `number[]` | 否 | 标签 ID 列表 |

### 响应体

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1001,
    "status": "DRAFT"
  }
}
```

### 业务规则
- 默认创建为 `DRAFT`
- 用户只能创建自己的记录

---

## 4.2 编辑草稿

### 接口
- `PUT /api/records/{id}`

### 请求体

```json
{
  "title": "给三个月后的自己",
  "content": "最近在准备实习，也在担心自己的能力。",
  "recordType": "NODE_RECORD",
  "coreQuestion": "我能不能慢一点也没关系",
  "unlockAt": "2026-06-01 09:00:00",
  "tagIds": [1, 6]
}
```

### 业务规则
- 只有 `DRAFT` 状态允许编辑
- 只能编辑自己的记录
- 已封存和已解锁记录不允许编辑

### 业务规则
- 只有 `DRAFT` 状态允许删除

---

## 4.4 封存记录

### 接口
- `POST /api/records/{id}/seal`

### 请求体

```json
{
  "unlockAt": "2026-06-01 09:00:00"
}
```

### 响应体

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1001,
    "status": "SEALED",
    "unlockAt": "2026-06-01 09:00:00",
    "sealedAt": "2026-03-16 10:20:00"
  }
}
```

### 业务规则
- 只有 `DRAFT` 状态允许封存
- `unlockAt` 必须晚于当前时间
- 封存后不可再编辑和删除

## 5. 回信模块接口

## 5.1 创建回信

### 接口
- `POST /api/records/{id}/reply`

### 请求体

```json
{
  "content": "你后来没有想象中那么糟，慢一点也可以。",
  "replyType": "SHORT_REPLY"
}
```

### 响应体

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 3001,
    "recordId": 1001,
    "content": "你后来没有想象中那么糟，慢一点也可以。",
    "replyType": "SHORT_REPLY",
    "createdAt": "2026-06-01 09:10:00"
  }
}
```

### 业务规则
- 只有记录本人可回信
- 只有 `UNLOCKED` 状态允许回信
- 同一记录只允许一条回信
- 回信创建后不可修改、不可删除

---

## 5.2 查询回信

### 接口
- `GET /api/records/{id}/reply`

### 响应体

```json
{

    "content": "你后来没有想象中那么糟，慢一点也可以。",
    "replyType": "SHORT_REPLY",
    "createdAt": "2026-06-01 09:10:00"
  }
}
```

### 业务规则
- 只能查看自己记录的回信
- 如果没有回信，可返回 `data = null`

## 7. AI 模块接口

AI 第一版不是核心主流程，所以接口要轻，且必须可降级。

## 7.1 写作提示

### 接口
- `POST /api/ai/writing-prompts`

### 请求体

```json
{
  "recordType": "NODE_RECORD",
  "coreQuestion": "我能找到合适的实习吗",
  "content": "最近总觉得自己做得不够好。"
}
```

### 响应体

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "prompts": [
      "你现在最担心的事情是什么？",
      "如果三个月后的你看到这段话，你最希望他回答什么？",

  }
}
```

### 业务规则
- AI 失败时返回兜底提示词
- 不允许直接生成整篇记录正文

## 10. Copilot 生成接口代码约束

### 10.1 路由约束
- 严格使用本文档定义的接口路径
- 不要擅自新增批量接口、导出接口、公开接口

### 10.2 DTO 约束
- 请求 DTO 和响应 VO 分离
- 列表查询参数统一收敛到 Query DTO
- 时间字段统一使用字符串或明确时间类型，不要混用时间戳

### 10.3 权限约束
- 所有 `/api/records/**` 接口都必须校验记录归属
- 所有 `/admin/**` 接口都必须校验管理员身份
- 不允许通过管理端接口修改用户记录正文

### 10.4 业务约束
- 编辑和删除只允许 `DRAFT`
- 回信只允许 `UNLOCKED`
- 同一记录只允许一条回信
- AI 接口失败时必须有兜底结果

## 11. 给 Copilot 的提问方式建议

建议按模块拆开提问，不要一次性生成全部接口。

### 示例 1：用户模块
`请根据接口清单文档实现 auth 和 user 模块的 Controller、DTO、VO、Service，接口路径和字段严格按文档，不要新增密码找回、邮箱验证码等超出 MVP 的功能。`

### 示例 2：记录模块
`请根据接口清单文档实现 records 模块，包含创建草稿、编辑草稿、删除草稿、封存记录、记录列表、记录详情、已解锁记录和时间轴接口，严格执行 DRAFT/SEALED/UNLOCKED 状态规则和记录归属校验。`

### 示例 3：回信模块
`请根据接口清单文档实现回信接口，要求只有 UNLOCKED 状态记录允许创建回信，同一 record_id 只允许一条回信。`

### 示例 4：AI 模块
`请根据接口清单文档实现 AI 写作提示和内容整理接口，先用 mock 方式返回固定结构，失败时提供兜底结果，不要实现复杂对话。`
